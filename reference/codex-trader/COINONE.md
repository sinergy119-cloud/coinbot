# COINONE

## 대상 계정

- 김재한
- 서수형
- 정명자
- 김지연
- 서영철

## 검증 상태

- `POST /v2.1/account/balance/all` 잔고 조회 검증 완료
- API 5개 모두 정상 응답 확인
- `coinone_api.ps1` 기본 스크립트 추가 완료
- Kakao 전송 경로 확인 완료

## 인증 및 호출 기준

- `X-COINONE-PAYLOAD`
- `X-COINONE-SIGNATURE`
- payload JSON은 base64 인코딩
- signature는 HMAC-SHA512 hex

## 현재 지원 기능

- 잔고 조회
- 주문 조회
- 거래내역 조회
- 시장가 매수
- 시장가 매도
- `market-cycle`
- Kakao 최종 결과 전송

## 코인원 이벤트 금액 규칙

- 코인원 이벤트 실적용 시장가 매수는 `5,000원`을 사용하지 않는다.
- 코인원 이벤트 실적용 시장가 매수는 `5,100원 이상`만 허용한다.
- 적용 대상:
  - `market-buy`
  - `market-cycle`
- 이유:
  - `5,000원` 주문은 실제 체결 총액 기준으로 `4,999원` 수준이 되면서 최소 주문 금액 조건에 걸릴 수 있다.
  - 이 경우 전량 매도 단계에서 최소 주문 금액 오류가 발생할 수 있다.

## 운영 메모

- 이벤트 실적용 요청에서 코인원이 포함되면 `5,000원`은 자동으로 불가 처리한다.
- 거래 요청 리스트에도 이 규칙을 반영해서 `불가능(코인원 이벤트 금액 제한)`처럼 안내한다.
- 최종 결과는 주문 2회 확인, 잔고 2회 확인 후 확정한다.
