# 공통 운영 규칙

## 1. 기본 파일

- API 원본: `코인 API 리스트.txt`
- 카카오 설정: `kakao_config.json`
- 문서에는 API Key, Secret Key를 직접 적지 않는다.

## 2. 결과 표시 규칙

- 거래 요청 리스트, 최종 결과, 잔고를 표로 정리한다.
- KRW 금액은 소수점 없이 표시한다.
- KRW 규칙은 잔고뿐 아니라 체결금액, 매수금액, 매도금액, 수수료 반영 금액에도 동일하게 적용한다.
- 통화 `P`는 `Point` 또는 `포인트`로 표시한다.
- 거래 요청 리스트의 불가 항목은 `불가능(사유)` 형식으로 표시한다.
- 최종 결과의 실패 항목은 `실패(사유)` 형식으로 표시한다.
- HTML 색상 표시는 기본 규칙에서 제외하고, 텍스트만으로도 바로 이해되게 작성한다.

## 3. 최종 결과 검증 규칙

- 주문 결과는 2회 이상 확인한다.
- 잔고는 2회 이상 확인한다.
- 같은 결과가 반복 확인될 때만 최종 결과로 확정한다.
- 값이 다르면 추가 확인 후 최종 결과를 확정한다.

## 4. 카카오 알림 규칙

- 카카오는 최종 확정 결과만 전송한다.
- 개별 주문마다 카카오를 여러 건 보내지 않는다.
- 카카오 설정은 공통 `kakao_config.json`을 사용한다.
- 실거래 실행 시 기본적으로 `-NotifyKakao`를 붙인다.
- 카카오 최종 요약은 `매수 51,000원 / 매도 50,403원` 형식으로 보낸다.
- 실패 사유는 `실패(잔고 부족)`처럼 짧고 읽기 쉽게 정리한다.

## 5. 거래소 추가 및 문서 업데이트 규칙

거래소가 추가되면 아래 순서로 같은 방식으로 업데이트한다.

1. `코인 API 리스트.txt`에 거래소 계정 정보 추가
2. `ACCOUNT_INDEX.md`에 거래소와 계정 상태 반영
3. `<EXCHANGE>.md` 문서 추가 또는 갱신
4. 잔고 조회와 거래 조회 방식 검증
5. 거래 스크립트와 Kakao 연동 여부까지 문서 반영

## 6. API 오류 대응 규칙

- API 오류가 나면 먼저 해당 거래소의 공식 API 문서를 기준으로 확인한다.
- 파라미터, 인증 방식, 헤더, 응답 형식은 공식 문서를 우선 기준으로 본다.
- 실제 동작과 공식 문서가 다르면 문서 확인 결과와 실제 결과를 구분해서 반영한다.

## 7. IP 사용 규칙

- 코인거래소 API 사용 시 기본 기준은 외부 공인 IP다.
- 화이트리스트 확인, 접속 오류 점검, 허용 여부 판단도 공인 IP 기준으로 진행한다.

## 8. 문서 작성 규칙

- 공통 규칙은 `COMMON_RULES.md`에 둔다.
- 거래소 고유 내용은 거래소별 MD에 둔다.
- 민감정보는 문서에 직접 적지 않는다.

## 9. 속도 개선 규칙

- 독립적인 잔고 조회는 기본적으로 병렬 처리한다.
- 독립적인 다중 계정 주문은 가능한 범위에서 병렬 처리한다.
- 같은 요청 안에서 중복되는 시세 조회, 계정 파싱, 거래내역 조회는 재사용한다.
- 거래내역 확인은 각 거래소 스크립트의 정식 `trades` 또는 `history` 명령을 우선 사용한다.

## 10. 코인원 이벤트 금액 규칙

- 코인원 이벤트용 시장가 매수는 `5,000원`을 차단한다.
- 코인원 이벤트용 시장가 매수는 `5,100원 이상`만 허용한다.
- 이유는 `5,000원` 주문이 실제 체결 총액 기준으로 `4,999원` 수준이 될 수 있어 최소 주문 금액 조건에 걸릴 수 있기 때문이다.
- 이 규칙은 코인원의 이벤트성 거래(`market-buy`, `market-cycle`)에 공통 적용한다.

## 11. 작업 완료 알림 규칙

- 작업 완료 후 기본적으로 윈도우 토스트 알림을 보낸다.
- 가능하면 `BurntToast` 모듈을 우선 사용하고, 불가능하면 내장 토스트/팝업으로 fallback 한다.
- 공용 알림 스크립트는 `C:\CODEX\notify_complete.ps1`이다.
- 알림은 CODEX 화면에 최종 메시지를 먼저 보여준 뒤, 약간 늦게 뜨도록 지연 실행한다.
- 기본 지연값은 `2초`다.
- 기본적으로 작업 시간 조건은 두지 않는다.
- 필요할 때만 `-MinElapsedSeconds`로 최소 작업 시간을 별도 줄 수 있다.
